
## Primary vs. Foreign Keys

A **primary key** is a column that uniquely identifies each row in a table:

- consider a table with many rows and columns, some of them having duplicate values, the column having unique values for each of the rows presented in the table, will be known as primary key.
- Think of it like a student ID number in a college database – no two students can have the same one.

A **foreign key**, on the other hand, is a column in one table that refers to the primary key in another table. Unlike primary key, it can have duplicate values. 🗝️
